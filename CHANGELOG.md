# 更新日志

本文档记录 ShareSDU 项目的版本更新历史。

## [未发布] - 开发中

### 性能优化
- **路由系统优化**（2026-03-06）
  - 实现平滑的页面切换过渡动画，消除空白闪烁
  - 优化异步组件加载延迟（delay=0），立即显示加载状态
  - 实现智能路由预加载机制，提升常用页面访问速度
  - 更新路由相关文档

## [1.5.0] - 最新版本

### 新增功能
- 增加了搜索 API 板块（技术栈：Elasticsearch + django-haystack）
- 增加了置顶操作相关板块
- 新增了邀请码相关 API
- 为帖子/文章获取列表和详情的 API 增加了置顶相关字段（`if_top`）
- 用户登录时返回管理员/超级管理员对应字段
- 为注册参数新增了邀请码相关字段

### 修复
- 修复了部分搜索字段的返回格式
- 修复了 ES 迁移字段时的 JVM 堆内存问题

### 技术改进
- Elasticsearch 配置优化：network.host 和禁用 https 认证

---

## [1.4.9]

### 新增功能
- 新增获取用户创作的 API，分为预览接口和详情接口
- 新增获取私信者和获取聊天记录的 API
- 用户列表添加分页查询功能
- 用户板块文档（包括主页，用户列表等）和账户板块分开

### 修复
- 修复了通知部分的 bug
- 修复了 signals 处理文章数量异常的 bug

### 改进
- **通知已读**支持批量修改，获取通知列表时不会包含已读信息
- 修订了下载 API
- 给几个 detail 接口新增了收藏量字段

---

## [1.4.8]

### 新增功能
- 图片审核功能正式上线
- 增加了**获取头像信息**的接口
- 分页获取文章**新增按热度排序的选项**

### 修复
- **修复了 refresh 接口的返回值**，status=999 时表示获取 access token 成功

### 改进
- 获取文章图片（即通用图片接口）新增 `audit_problems` 返回字段，表示审核未通过的原因
- access token 过期时间延长到 5h
- 修改了文章板块的部分参数，返回时增加 `hot_score` 字段
- 完善了 article 的删除接口
- 为部分接口字段增加了 author_id/poster_id/replier_id/receiver_id/sender_id
- 修改了获取收藏内容，**修改了返回格式，支持分页获取**
- 完善取消收藏的接口，增加 404 检查
- 课程详情接口**增加了 `score_distribution` 字段**
- 更新了**创建 reply**的接口

---

## [1.4.7]

### 新增功能
- **增加了分页获取 tag 信息的 API**
- 简单补充了**课程管理部分的 API 文档**（冻结、解冻、版本回滚、获取课程历史版本等）
- 新增 systemctl 配置和**校对用的路由配置**
- 新增无冻结检查获取图片的 API 用于审核

### 改进
- 为 Article 和 Tag 增加了**热度**相关字段
- article 的 detail 多了 `article_summary` 和 `hot_score` 字段
- 用信号量重构了部分原子+1 的逻辑
- 基于 celery 的定时热度更新，分片操作
- 因为设置了过期 token 的异步清理，所以**状态码 1000 的时候也要刷新一下**
- 登陆返回应该有 refresh 字段，已添加
- 优化了部分信号任务，对部分 list 获取增加了缓存
- 优化了编辑 article/获取 article 详情的部分（增加 cover_link 字段作为封面链接）
- 做了图片审核的部分内容
- 修正了部分返回的状态码
- 修正了**管理课程版本部分的 API**

### 修复
- 修复了多线程问题下 celery 通知多次触发的问题

## [1.4.6]

### 改进
- 把 block 检查放在自定义认证类里了
- 修改了线程操作，支持敏感词库热更新
- 修改了用户和文章 block 板块的 API（**重命名了部分 API**，修改了权限设置，**增加了解除文章屏蔽的 API**）
- 管理员屏蔽/解除屏蔽后会对用户进行通知（自动解封不会通知）
- 被 block 的文章在分页获取文章列表时不会被选中
- 更新了通知类型，增加了 `system message` 选项
- 重构了用户板块（**API 不变**），邮件发送改为 celery 异步操作
- **重构了自定义 JWT 认证板块**，**更新了该板块规范的状态码**
- **base_url 改过来了（即：把所有的 /index/ 替换成了 /index/api/）**

## 版本说明

- 版本号格式：主版本号.次版本号.修订号
- 主版本号：重大架构变更或不兼容的 API 修改
- 次版本号：新增功能，向下兼容
- 修订号：问题修复和小的改进
